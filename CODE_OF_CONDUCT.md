# Code of Conduct

The `shieldops-sdk` project is an open community of contributors, operators,
and security researchers. We want participation to feel safe and worthwhile
for everyone, regardless of background or experience.

This project follows the **Contributor Covenant Code of Conduct, version 2.1**:
<https://www.contributor-covenant.org/version/2/1/code_of_conduct/>.

The full text is hosted at that link rather than copied here so that you are
always reading the canonical, up-to-date version. By participating in this
project — in issues, pull requests, discussions, or any other channel
associated with the repository — you agree to abide by it.

## Reporting

If you experience or witness behavior that violates the Code of Conduct,
please report it to **conduct@shieldops.io**. Reports are read by the
project maintainers (see [MAINTAINERS.md](MAINTAINERS.md)) and are treated
as confidential. Include as much context as you are comfortable sharing —
links to messages, approximate timestamps, and the people involved are all
helpful but not required.

If a report concerns a maintainer, send it to **conduct@shieldops.io** with
a note that it is maintainer-related; reports about a maintainer are
reviewed by the other maintainers, not the subject of the report.

## Enforcement

Maintainers are responsible for clarifying and enforcing the Code of
Conduct, and may take any action they deem appropriate, up to and including
warning, temporary suspension, or permanent removal from project spaces.
The Contributor Covenant 2.1 enforcement guidelines (linked above) describe
the escalation ladder we use.

For security vulnerabilities — distinct from conduct issues — please follow
[SECURITY.md](SECURITY.md) instead.
